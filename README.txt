============================================================
  在庫管理システム v27
  セットアップ・操作ガイド
============================================================

【対応環境】
  OS  : Windows 10 / 11 / Server 2019 / 2022
  Python: 3.10 以上（3.14推奨）
  DB  : PostgreSQL（同一PC または ネットワーク上のサーバー）

------------------------------------------------------------
【初回セットアップ手順】
------------------------------------------------------------

1. Python のインストール
   https://www.python.org/downloads/
   ※ インストール時「Add Python to PATH」にチェックを入れること

2. PostgreSQL のインストール（ローカルDBを使用する場合）
   https://www.postgresql.org/download/windows/
   ※ すでに別サーバーにDBがある場合はスキップ

3. setup.bat を実行（ダブルクリック）
   以下を自動で行います:
   ・Python のバージョン確認
   ・必要パッケージのインストール
   ・.env ファイルの対話的生成（DB接続情報・ポートを入力）
   ・データベース初期化・テーブル作成

4. 起動確認
   ブラウザで http://localhost:5000 を開く
   ユーザー名: admin / パスワード: admin でログイン

------------------------------------------------------------
【各ファイルの説明】
------------------------------------------------------------

  setup.bat            ★ 初回セットアップ（最初に実行）
  start.bat            通常起動（ダブルクリックで起動）
  stop.bat             停止
  install_service.bat  Windowsサービス登録（管理者で実行）
  uninstall_service.bat サービス削除（管理者で実行）
  service_log.bat      ログ確認（最新50行を表示）
  .env.example         設定ファイルのサンプル

------------------------------------------------------------
【Windowsサービスとして登録する場合（推奨）】
------------------------------------------------------------

  1. nssm\nssm.exe を配置（README参照）
  2. install_service.bat を右クリック → 管理者として実行
  → PC再起動後も自動起動されます

------------------------------------------------------------
【ファイル更新後の再起動手順】
------------------------------------------------------------

  ① taskkill /f /im python.exe
  ② rd /s /q __pycache__
  ③ start.bat

------------------------------------------------------------
【トラブルシューティング】
------------------------------------------------------------

  ▶ 起動しない場合
    service_log.bat を実行してエラーを確認

  ▶ パッケージエラーの場合
    コマンドプロンプトで:
    python -m pip install -r requirements.txt

  ▶ DB接続エラーの場合
    .env のPG_HOST・PG_PORT・PG_PASSWORD を確認

  ▶ ポートが使えない場合
    .env の PORT=5000 を別の番号に変更（例: 8080）
