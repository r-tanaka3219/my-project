## 1. ログイン・基本操作

### ログイン

1. ブラウザで `http://サーバーアドレス:5000` にアクセス
2. ユーザー名・パスワードを入力して「ログイン」をクリック
3. 初回ログイン時はパスワード変更が求められます

### 画面構成

- **ヘッダー左**: システム名・ナビゲーションメニュー
- **ヘッダー右**: ログインユーザー名・ログアウトボタン
- **メインエリア**: 各機能の操作画面

### テーブルのソート

すべての画面で、テーブルのヘッダー行をクリックするとその列で並べ替えができます。
- 1回クリック: 昇順（▲）
- 2回クリック: 降順（▼）

## 3. 商品管理

URL: `/products`

### 商品一覧

- 登録済み全商品を表示
- 検索バーで商品名・JAN・商品コードで絞り込み
- 「非アクティブ商品」ボタンで休止中商品を確認

### 商品新規登録

1. 「＋ 新規登録」ボタンをクリック
2. 以下の項目を入力:

| 項目 | 説明 |
|------|------|
| 仕入先CD | 仕入先の識別コード |
| 仕入先名 | 仕入先会社名 |
| JANコード | 13桁または8桁のバーコード |
| 商品コード | 仕入先の商品コード |
| 商品名 | 商品の名称 |
| 入数 | 1ケース内のピース数 |
| 発注単位 | case（ケース）またはpiece（本） |
| 発注点 | この在庫数量を下回ったら発注 |
| 発注数 | 1回の発注数量 |
| ロット数 | 最小発注ロット（複数ロット時の倍数） |
| リードタイム | 発注から入庫までの日数 |
| 賞味期限日数 | 商品の標準賞味期限 |
| 期限アラート日数 | 期限何日前にアラートを出すか |
| 安全係数 | 安全在庫の余裕率（1.0=係数なし） |
| 原価 | 仕入単価 |
| 売価 | 販売単価 |

3. 「保存」をクリック

### 発注パラメータの意味

- **発注点**: 在庫がこの数量を下回ると自動発注が起きます
- **発注数**: 1回の発注で何個注文するか
- **ロット数**: 発注数はこの値の倍数に切り上げられます
- **リードタイム**: 発注後、商品が届くまでの日数。発注点計算に使います

### 混合グループ設定

複数の商品を同じグループIDに設定すると、グループ内のどれかが発注点以下になったときに、グループ全体を1つのロットとしてまとめて発注します。

### 季節品管理

- **season_start_mmdd** / **season_end_mmdd**: `04-01` 形式で販売期間を設定
- 期間外は自動的に非アクティブになり発注が止まります
- 「季節品の有効化/無効化」ボタンで手動一括切替も可能

### 商品の一括インポート

1. 「テンプレートDL」でExcelテンプレートをダウンロード
2. テンプレートに商品情報を入力
3. 「インポート」ボタンからファイルをアップロード

## 4. 在庫管理

URL: `/inventory`

### 在庫一覧

- 商品ごとの全ロット在庫を表示
- 賞味期限・ロット番号・保管場所が確認できます
- 期限切れ近いロットは背景色で警告表示

### ロット廃棄

1. 廃棄するロットの「廃棄」ボタンをクリック
2. 廃棄理由・メモを入力して確定
3. 廃棄記録が保存され在庫数から差し引かれます

### 廃棄済み在庫の復元

URL: `/inventory/disposed`

誤って廃棄した場合は廃棄済み一覧から「復元」できます。

### ロット振替

在庫を別の棚・場所へ移動する場合に使用します。

### ピッキングリスト

URL: `/inventory/picking`

出荷・補充用のピッキングリストをCSV・Excel・印刷用紙で出力できます。

### 棚補充管理

URL: `/inventory/replenishment`

販売棚への補充タスクを管理します。

## 5. 発注管理

### 保留中発注の確認

URL: `/orders`

自動発注エンジンが生成した発注候補が「保留中」として表示されます。

| 列 | 説明 |
|----|------|
| 仕入先 | 発注先の仕入先名・CD |
| 商品 | JAN・商品コード・商品名 |
| 発注数 | 発注する数量 |
| 発注理由 | auto（自動）/ manual（手動）等 |
| 保留期間 | 保留してからの経過日数 |

**操作:**
- **送信**: チェックした発注をメールで送信し、発注履歴へ移動
- **キャンセル**: 不要な保留発注を削除
- **グループ発注**: 同一仕入先をまとめて確認・調整してから送信

### 未到着発注の管理

URL: `/orders/backorders`

発注済みでまだ入庫していない発注を追跡します。

- **受入**: 入庫確認が取れた場合にクリック
- **クローズ**: キャンセル・入手不可の場合に使用

### 発注履歴

URL: `/order_history`

過去の全発注記録を確認できます。メール送信結果も表示されます。

## 6. 入庫登録

URL: `/receipt`

仕入先から商品が届いたときに入庫登録を行います。

1. JANコードまたは商品名で商品を検索・選択
2. 入庫数量・ロット番号・有効期限を入力
3. 「登録」をクリック

### 入庫履歴

URL: `/receipt/history`

過去の入庫記録を確認・削除できます。

## 7. CSV取込

URL: `/csv`

### CSV設定の作成

1. 「＋ 新規設定」をクリック
2. 以下を設定:

| 設定項目 | 説明 |
|---------|------|
| 設定名 | 識別用の名前（例: 〇〇チェーン日次） |
| 取込種別 | daily（日次）/ monthly（月次） |
| フォルダパス | CSVファイルの保存場所 |
| ファイル名パターン | 例: `sales_{YYYYMMDD}.csv`（日付プレースホルダ使用可） |
| エンコーディング | auto / utf-8 / shift_jis |
| JANコード列 | CSVのJANコードが入っている列名または列番号 |
| 数量列 | 販売数量の列 |
| 日付列 | 売上日の列 |
| チェーンCD列 | チェーンコードの列（任意） |
| 担当CDフィルター | 特定の担当CDのみ取込む場合に設定 |
| 実行時刻 | 自動実行する時刻（複数設定可） |

**ファイル名プレースホルダ例:**
- `{YYYY}` → 年（2026）
- `{MM}` → 月（04）
- `{DD}` → 日（05）
- `{YYYYMMDD}` → 年月日（20260405）

### 手動実行

1. 設定一覧の「▶ 実行」ボタンをクリック
2. 対象ファイルを選択して実行
3. 進捗モニター画面でリアルタイム進捗を確認

**進捗モニターの見方:**
- **成功**: 正常に取込まれた行数
- **スキップ**: 重複（既に取込済み）でスキップされた行数
- **エラー**: 取込に失敗した行数

### 月末月次インポート

月末締め後に月次CSVを取り込む場合は「月末月次」ボタンを使用します。

### インポート詳細

URL: `/csv/import_detail`

取込済みの全ファイル一覧と、ファイル単位での件数・エラーを確認できます。
再インポートや誤ったファイルの削除もここから行います。

## 8. 需要予測（AIモード）

URL: `/reports/forecast/wholesale`

### 画面の見方

各商品の予測結果を表形式で表示します。

| 列 | 説明 |
|----|------|
| ABCランク | A/B/C（売上金額ベースの重要度） |
| 消化日数 | 現在在庫が何日分か（在庫÷日次Q50） |
| 30日予測 | 今後30日間の予測販売数量（Q50×30） |
| Q25/Q50/Q75 | 日次販売の下位25%/中央値/上位75% |
| 動的安全在庫 | AI計算による適正安全在庫数 |
| 気温補正 | 気温感応度による補正係数 |
| 推奨発注点 | AIが推奨する発注点 |
| 推奨発注数 | AIが推奨する1回あたり発注数 |
| 欠品日数 | 過去30日の欠品（売上ゼロ）日数 |
| IQR除外 | 外れ値として除外されたデータ日数 |
| MAPE | 予測精度（低いほど精度が高い） |

### 発注点への反映

1. 「推奨値を発注点に反映」ボタンをクリック
2. 反映対象を確認（ロック商品は除外）
3. 確認後「反映実行」をクリック
4. 進捗表示で完了を確認

### 予測モード

各商品には3つの予測モードがあります:

| モード | 説明 |
|--------|------|
| AI（推奨） | ホルト・ウィンタース予測を使用 |
| 前年実績 | 前年同期の実績値を使用 |
| 手動 | 管理者が設定した固定値を使用 |

「設定 > 予測モード一括切替」で全商品を一括で変更できます。

### 販促計画

URL: `/reports/forecast/wholesale`（画面下部）

特売・プロモーション期間中は需要が増加します。販促計画を登録することで予測数量に乗数（uplift_factor）が適用されます。

### 需要計画

スポット需要（特定得意先からの大口注文予定など）を日付・数量で登録できます。

## 9. 52週MDプラン

URL: `/reports/weekly_md`

### 概要

1年52週の週次販売計画（目標数量）と実績を管理します。

### セルの見方

```
     5    ← 計画数（週の販売目標数量）
     0    ← 実績数（sales_historyからの実際の販売数）
    0%    ← 達成率（実績÷計画×100）
  予27    ← AI週次予測数（現在週のみ表示）
   +22    ← 予測が計画を上回る差分（緑）
    -5    ← 計画が予測を超えている差分（赤）
```

**達成率の色:**
- 緑（≥100%）: 達成
- 橙（≥70%）: 要注意
- 赤（<70%）: 未達

**背景色:**
- 黄色: 現在週

### 計画数の編集

計画数（青い数字）をクリックすると直接入力できます。
- Enter または フォーカスを外す: 保存
- Esc: キャンセル

### 計画の一括生成

「前年実績から一括生成」ボタンで前年同期の実績数量を計画数として自動設定します。

### 発注数への反映

「発注数へ反映」ボタンを押すと、今後4週の計画数から発注数・発注点を自動計算して更新します（ロック商品は除外）。

## 12. 気象データ管理

URL: `/reports/weather`

### 気象データの取得方法

**方法1: 気象庁APIから自動取得**
1. 「気象データ取得」ボタンをクリック
2. 地点を選択（.envの`WEATHER_LOCATION`で設定）
3. 「取得実行」をクリック

**方法2: CSVインポート**
1. 「テンプレートDL」でCSVテンプレートをダウンロード
2. 日付・平均気温・最高気温・最低気温・降水量を入力
3. 「CSVインポート」からアップロード

**方法3: Excelインポート**
同様の手順でExcel形式でもインポート可能。

### 気温感応度の再計算

気象データを更新・追加した後は「気温感応度を再計算」ボタンを押してください。
商品マスタに登録済みのJANコードのみ対象に計算します。

## 13. 棚卸

URL: `/stocktake`

### 棚卸の流れ

1. 「棚卸を作成」ボタンをクリック（システム在庫数を取込）
2. 実際の棚の在庫数を数えて「実棚数」欄に入力
3. 差分（システム数 - 実棚数）を確認
4. 差分理由を選択・入力
5. 「棚卸確定」で在庫数を実棚数に調整

### エクスポート

棚卸結果をCSVまたはExcelで出力できます。

## 18. バックアップ・復元

URL: `/settings`（管理者のみ）

### メインバックアップ（Excel）

「バックアップ」ボタンをクリックするとExcelファイルがダウンロードされます。

含まれるデータ:
- 商品マスタ
- 在庫ロット・廃棄在庫
- 発注履歴・入庫記録
- CSV設定・メール宛先・ユーザー

### 売上履歴バックアップ（CSV）

売上データのみを別途CSVでバックアップします。

**エクスポート手順:**
1. 「全件CSVダウンロード」ボタンをクリック
2. 進捗モーダルで生成状況を確認（件数・パーセント表示）
3. 生成完了後、自動でダウンロードが始まります

**復元手順:**
1. 「ファイルを選択」でエクスポートしたCSVを選択
2. 「復元実行」をクリック
3. 重複データは自動スキップされます（ROW_HASHによる照合）
4. 「N件追加 / M件スキップ」の結果が表示されます

> ※ 復元できるのは、このシステムでエクスポートしたCSVのみです

### データ復元（Excelバックアップから）

URL: `/admin/restore`

1. バックアップExcelファイルを選択
2. 復元したいデータ（商品マスタ・在庫・発注履歴等）にチェック
3. 「復元実行」をクリック

## 19. 自動処理スケジュール

以下の処理が自動で定期実行されます。

| 実行タイミング | 処理内容 |
|--------------|---------|
| 毎日 AM6:00 | 日次CSVインポート（全アクティブ設定） |
| 毎日 AM8:00 | 発注点チェック・在庫引当・アラートメール送信 |
| 毎月末日 AM5:00 | 月末月次CSVインポート |
| 毎月1日 AM6:00 | AI予測による発注点自動更新 |
| 毎月15日 | 気象データ自動取得 |
| 毎年1月1日 | 52週MD計画の新年度生成 |

実行時刻は `.env` の `DAILY_MAIL_HOUR` / `DAILY_MAIL_MINUTE` で変更できます。

## 20. トラブルシューティング

### CSV取込でファイルが見つからない

- フォルダパスを確認してください（UNCパス: `\\サーバー名\共有名\`）
- ファイル名パターンの日付プレースホルダが正しいか確認
- ネットワークドライブの場合は「ネットワーク接続設定」を確認

### 発注メールが送信されない

1. 「設定」→「メール送信テスト」で接続確認
2. SMTPサーバー・ポート・認証情報を確認
3. メール宛先に有効なアドレスが登録されているか確認

### 予測値がおかしい

- 売上データが十分あるか確認（最低30日分推奨）
- ABCランクを確認（Cランクは単純移動平均になります）
- 外れ値が多い場合は「IQR除外件数」列を確認
- 気温補正が効いている場合は感応度を確認

### バックアップが完了しない

- 売上データが大量の場合は時間がかかります。進捗モーダルで件数を確認してください
- ブラウザを閉じても処理は継続されます（ただし進捗は確認できません）

### 「在庫管理システム」にログインできない

- パスワードを3回以上間違えるとロックされる場合があります
- 管理者に連絡してアカウントの確認を依頼してください

### 気温感応度の再計算件数が多すぎる

商品マスタに登録されている商品のみが対象です。`sales_history` に存在するが商品マスタに未登録のJANは計算されません。件数が少ない場合は正常です。
