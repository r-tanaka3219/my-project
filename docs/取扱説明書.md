## 1. はじめに

### 1.1 本書について

本書は在庫管理・自動発注システムの操作方法を説明します。

### 1.2 画面共通操作

| 操作 | 方法 |
|---|---|
| 検索 | 検索ボックスにキーワードを入力して「検索」をクリック |
| クリア | 「クリア」ボタンをクリック |
| データ保存 | フォーム入力後「保存」ボタンをクリック |
| エクスポート | 「CSV出力」または「Excel出力」ボタンをクリック |
| 削除 | 「削除」ボタンをクリック → 確認ダイアログで「OK」 |

### 1.3 色の意味

| 色 | 意味 |
|---|---|
| 赤色 | 警告・エラー・発注点以下 |
| 黄色 | 注意・期限間近 |
| 緑色 | 正常・完了・在庫充分 |
| 青色 | 情報・リンク |

## 2. ログイン・初期設定

### 2.1 ログイン手順

1. ブラウザで `http://[サーバーアドレス]:[ポート]` を開く
2. ユーザー名とパスワードを入力
3. 「ログイン」をクリック

**初期アカウント**:

| ユーザー名 | 初期パスワード | ロール |
|---|---|---|
| admin | admin123 | 管理者（全機能） |
| user | user123 | 一般ユーザー |

> 重要: 初回ログイン時にパスワードの変更が求められます。必ず変更してください。

### 2.2 初期設定チェックリスト

新規導入時は以下の順序で設定してください。

```
□ 1. システム設定 → PostgreSQL接続設定を確認
□ 2. システム設定 → メールサーバー設定を入力
□ 3. システム設定 → 自動送信スケジュールを設定
□ 4. ユーザー管理 → 利用者アカウントを作成・権限設定
□ 5. メール受信者 → 発注メール送信先を登録
□ 6. 商品管理 → 商品マスタをインポート
□ 7. 在庫管理 → 初期在庫を入庫処理で入力
□ 8. CSV取込設定 → 売上CSVの取込設定を作成
□ 9. 動作確認 → テストメール送信で疎通確認
```

## 5. 在庫管理

**URL**: `/inventory`
**権限**: inventory

### 5.1 在庫一覧

在庫はJAN・賞味期限・ロット番号単位で表示されます。

**表示カラム**:

| カラム | 説明 |
|---|---|
| 仕入先CD | 商品の仕入先 |
| 商品CD / JAN | 商品識別コード |
| 商品名 | 商品名称 |
| 在庫数 | 現在の在庫個数 |
| 賞味期限 | 賞味期限日 |
| ロット番号 | ロット識別番号 |
| ロケーション | 保管場所コード |

**在庫一覧サンプル**:

| 仕入先CD | JAN | 商品名 | 在庫数 | 賞味期限 | ロット | 操作 |
|---|---|---|---|---|---|---|
| SUP001 | 4901234567890 | 緑茶 500ml | 144 | 2026-08-31 | L240101 | 編集 廃棄 移動 |
| SUP001 | 4901234567890 | 緑茶 500ml | 72 | 2026-06-30 | L240215 | 編集 廃棄 移動 |
| SUP002 | 4908765432100 | 麦茶 50袋 | 96 | 2027-03-31 | L240301 | 編集 廃棄 移動 |

### 5.2 在庫直接編集

「編集」ボタンをクリックすると数量・賞味期限・ロット番号を直接変更できます。

> 在庫の直接編集は移動履歴に記録されます。

### 5.3 廃棄登録

1. 廃棄する在庫行の「廃棄」ボタンをクリック
2. 廃棄数量・廃棄理由を入力
3. 「廃棄実行」をクリック

廃棄した商品は「廃棄・退避一覧」に移動し、在庫から差し引かれます。

### 5.4 入庫管理

**URL**: `/receipt`

商品の入荷時に入庫登録を行います。

1. 「入庫登録」をクリック
2. JAN（またはAPIで商品検索）・数量・賞味期限・ロット番号を入力
3. 「登録」をクリック

> 在庫にない商品（新JAN）の場合は自動的に新規在庫として登録されます。

### 5.5 棚補充管理

**URL**: `/inventory/replenishment`

倉庫の在庫を陳列棚に補充する作業を管理します。

1. 「補充計画を作成」で対象商品と補充数量を指定
2. 作業者が実際に補充を実施
3. 「完了」ボタンで補充実績を記録

## 7. CSV取込設定

**URL**: `/csv`
**権限**: csv

### 7.1 CSV設定の作成

1. 「新規設定を作成」をクリック
2. 各項目を設定して保存

**設定項目**:

| 項目 | 説明 | 例 |
|---|---|---|
| 設定名 | 管理用の名前 | チェーンA日次取込 |
| フォルダパス | CSVが保存されるフォルダ | \\\\192.168.1.10\\sales |
| ファイル名パターン | 変数を使ったファイル名 | sales_{yyyymmdd}.csv |
| 文字コード | CSVの文字コード | Shift-JIS |
| JAN列番号 | JANコードが入っている列（1始まり） | 3 |
| 数量列番号 | 売上数量が入っている列 | 5 |
| 日付列番号 | 売上日が入っている列 | 1 |
| 実行時刻 | 自動取込の時刻（HH:MM） | 08:00, 20:00 |

### 7.2 手動実行

1. CSV設定一覧で対象設定の「実行」をクリック
2. 対象日付を入力（省略時は当日）
3. 「実行」をクリック
4. 進捗バーで取込状況を確認

### 7.3 取込結果の確認

「インポート明細・確認」画面で取込結果の詳細を確認できます。

| ステータス | 意味 |
|---|---|
| success | 正常完了 |
| partial_skip | 一部スキップあり（未登録JAN等） |
| error | エラーが発生した |
| skip | ファイルなし等でスキップ |

**未登録JANの確認**:
`partial_skip` の行をクリックすると、マスタ未登録のJANコード一覧が表示されます。該当JANを商品マスタに登録してから再取込を行ってください。

## 8. AI需要予測

**URL**: `/reports/forecast/wholesale`
**権限**: forecast

### 8.1 予測一覧の見方

| 列名 | 説明 |
|---|---|
| ABCランク | A=重点商品・B=準重点・C=その他 |
| 30日予測 | 今後30日の予測販売数 |
| Q50日次 | 中央値（毎日の予測数量） |
| 動的安全在庫 | IQRベースの安全在庫数 |
| 推奨発注点 | AIが計算した発注点 |
| 推奨発注数 | AIが計算した発注数量 |
| 気温補正係数 | 気温による補正倍率（1.0=補正なし） |

### 8.2 予測を発注設定に反映する

1. 反映したい商品にチェックを入れる（または全選択）
2. 反映モードを選択
   - 「発注点のみ更新」: 推奨発注点のみ反映
   - 「発注数のみ更新」: 推奨発注数のみ反映
   - 「両方更新」: 発注点・発注数の両方を更新
3. 「予測を発注設定に反映」をクリック

### 8.3 予測モードの切り替え

「システム設定」→「需要予測ロジック設定」で切り替え可能：

| モード | 説明 | 推奨状況 |
|---|---|---|
| AI予測モード（ON） | ホルト・ウィンターズ + 分位点 + 気温補正 | 売上データが90日以上蓄積後 |
| 前年実績モード | 前年同月の日次平均を使用 | 導入初期・売上データ少量時 |

## 9. ABC分析

**URL**: `/reports/abc`
**権限**: forecast

### 9.1 ABC分析の見方

過去365日の売上金額に基づき商品をA・B・C・Xに分類します。

| ランク | 基準（デフォルト） | 対応方針 |
|---|---|---|
| A | 売上上位70%を占める商品 | 欠品絶対回避・AI予測を積極活用 |
| B | 売上上位70〜90%を占める商品 | 標準的な在庫管理 |
| C | 残り10%の商品 | 在庫圧縮・定期発注を検討 |

### 9.2 XYZ分析（変動係数）

| ランク | 変動係数 | 意味 |
|---|---|---|
| X | 0.25未満 | 需要が安定している |
| Y | 0.25〜0.75 | やや変動がある |
| Z | 0.75以上 | 需要が不規則 |

**AZ商品**（Aランク×Z分析）: 重要商品だが需要が不安定 → 特に注意が必要

## 10. 52週MDプラン

**URL**: `/reports/weekly_md`
**権限**: forecast

### 10.1 MDプランの作成

**前年実績から自動生成する場合**:
1. 「前年実績から一括生成」をクリック
2. 年度を選択して確定

**手動で計画数を入力する場合**:
1. 対象商品の週セルをクリック
2. 計画数を入力してEnter

### 10.2 表の見方

各セルの表示：

```
  5    ← 計画数（週の販売目標数量）
  3    ← 実績数（実際の販売数量）
 60%   ← 達成率（実績÷計画×100）

色の意味:
  緑  >= 100% 達成
  橙  >=  70% もう少し
  赤  <   70% 未達
  黄色背景    現在週
```

### 10.3 発注への反映

今後4週分のMDプラン計画数をもとに発注数・発注点を自動計算して更新します。

1. 「発注数へ反映」をクリック
2. ロック設定のある商品は除外されます

## 12. 棚卸

**URL**: `/stocktake`
**権限**: stocktake

### 12.1 棚卸の手順

```
Step 1: 棚卸リスト作成
  └─ 対象日付を入力して「棚卸リストを作成」をクリック

Step 2: 実棚数の入力
  └─ 各商品の実際の在庫数を入力欄に入力

Step 3: 保存
  └─ 「実棚数を保存」をクリック（何度でも保存可）

Step 4: 差異の確認
  └─ システム在庫と実棚数の差異を確認する

Step 5: 在庫への反映（取消不可）
  └─ 「差異を在庫に反映」をクリック
  └─ 確認ダイアログで「OK」をクリック
```

> 「差異を在庫に反映」は取消できません。必ず差異内容を確認してから実行してください。

### 12.2 棚卸シートのエクスポート

- 「CSV出力」: CSV形式
- 「Excel出力」: 印刷に適したExcel形式
- 「印刷」: ブラウザの印刷機能で直接印刷

## 13. 気温データ管理

**URL**: `/reports/weather`
**権限**: forecast

### 13.1 気象データ取得（API）

1. 取得したい地点にチェックを入れる（複数選択可）
2. 「取得日数」を選択（7/14/30/60/92/180/365日）
3. 「選択地点を一括取得」をクリック
4. 進捗バーで取得状況を確認

> 全都道府県 × 365日の場合、約10〜15分かかります。

### 13.2 Excelインポート

過去データ（92日以上前）をExcelファイルで一括インポートできます。

1. 「テンプレートDL」でテンプレートをダウンロード
2. 以下の列にデータを入力:

| 列 | 内容 | 例 |
|---|---|---|

## 17. バックアップ・復元

### 17.1 バックアップ（エクスポート）

**URL**: `/settings`（設定画面のバックアップセクション）

「バックアップをダウンロード」をクリックすると、以下データを含むExcelファイルがダウンロードされます：

- 商品マスタ
- 在庫一覧
- 発注履歴
- 廃棄退避在庫
- 入庫履歴
- 棚卸データ

**ファイル名**: 在庫管理システム_バックアップ_YYYY-MM-DD.xlsx

### 17.2 売上履歴バックアップ

売上履歴は件数が多いため、別途CSVでバックアップします。

1. 設定画面の「全件CSVダウンロード」をクリック
2. 進捗バーで生成状況を確認
3. 完了後「ダウンロード」ボタンが表示される

### 17.3 復元手順

**URL**: `/admin/restore`

1. 「バックアップファイルを選択」でExcelファイルをアップロード
2. 復元したい対象にチェックを入れる
3. 「復元実行」をクリック
4. 確認ダイアログで「OK」をクリック

> 商品マスタ以外は復元前に既存データが全削除されます。

### 17.4 データ一括削除

**URL**: `/admin/bulk_delete`

> 削除したデータは復元できません。必ずバックアップを取得してから実行してください。

## 18. よくある質問（FAQ）

### Q1. 発注メールが届かない

**確認事項**:
1. 設定 → メールサーバー設定が正しいか確認
2. 「テストメール送信」で疎通確認
3. メール受信者に正しいメールアドレスが登録されているか確認
4. 受信者の「有効」フラグがONになっているか確認
5. 仕入先CDが商品マスタに設定されているか確認

### Q2. CSVが自動取込されない

**確認事項**:
1. CSV取込設定の「有効」フラグがONになっているか確認
2. フォルダパスが正しいか確認（手動実行でテスト）
3. ファイル名パターンが実際のファイル名と一致するか確認
4. ネットワーク共有の場合、認証情報が正しいか確認
5. 設定された実行時刻が正しいか確認

### Q3. 在庫数が実際と合わない

**対処法**:
1. 棚卸機能で実棚カウントを実施
2. 差異を確認後「差異を在庫に反映」で修正
3. 廃棄登録や在庫直接編集で個別修正も可能

### Q4. AI予測の精度が低い

**確認事項と対処法**:
1. 売上データが90日以上蓄積されているか確認
2. データが少ない場合は「前年実績モード」に切り替える
3. 気温補正を活用する場合は気温データを取得する
4. 異常値（特売・イベント）の影響を手動調整係数で補正

### Q5. 商品が予測一覧に表示されない

**確認事項**:
1. 商品マスタで「アクティブ」になっているか確認
2. 売上履歴テーブルに対象JANのデータがあるか確認
3. 検索フィルターが設定されていないか確認

### Q6. ログインできない

**確認事項**:
1. ユーザー名・パスワードを確認
2. ユーザー管理でアカウントが「有効」になっているか確認
3. 10回連続失敗した場合は1分間ロックされます（待ってから再試行）
