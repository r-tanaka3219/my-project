## 3. 主要機能仕様

### 3.1 需要予測エンジン

#### ABCランク分類

```
売上金額（cost_price × quantity）を商品ごとに合計
↓
全商品の売上金額合計に対する各商品の累積比率を計算
↓
累積比率 ≤ 70%  → Aランク（重点管理）
累積比率 ≤ 90%  → Bランク（標準管理）
累積比率 > 90%  → Cランク（簡易管理）
```

#### 予測アルゴリズム（Aランク）

ホルト・ウィンタース指数平滑法（加法型）を使用:

```
各パラメータ:
  α（レベル平滑化）= 0.3
  β（トレンド平滑化）= 0.1
  γ（季節平滑化）= 0.2
  季節周期 = 7日（週次パターン）
```

#### 外れ値除去（IQR法）

```
Q1 = PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY qty)
Q3 = PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY qty)
IQR = Q3 - Q1
除外条件: qty < Q1 - 1.5×IQR  OR  qty > Q3 + 1.5×IQR
```

#### 動的安全在庫計算

```
σ_AI = (Q75_daily - Q25_daily) / 1.349  （IQR→標準偏差換算）
Z = 1.65  （サービスレベル95%）
L = リードタイム（日）
動的安全在庫 = Z × σ_AI × √L
```

#### 推奨発注点・推奨発注数

```
推奨発注点 = Q50_daily × L + 動的安全在庫
推奨発注数 = Q50_daily × (L + サイクル期間)
```

#### 気温感応度計算

```
過去365日の日次売上と日次平均気温でOLS線形回帰
temp_coef = 気温1℃あたりの売上変化量
補正係数 = 1 + temp_coef × (予測気温 - base_temp) / base_daily
```

### 3.3 CSV取込フロー

```
1. ファイル名パターン解析（{YYYY}/{MM}/{DD}等のプレースホルダを実日付に展開）
2. ネットワークパス接続（net use 使用時）
3. ファイル検索（正規表現パターンマッチ）
4. エンコーディング自動判定（utf-8-sig → utf-8 → cp932 の順で試行）
5. 列指定に従いJAN・数量・日付・チェーンCD等を抽出
6. 担当CDフィルター適用（設定時）
7. 行ハッシュ計算（SHA-256）
8. 重複チェック（row_hash UNIQUE制約）
9. sales_history へ UPSERT
10. sales_daily_agg へ集計反映
11. インポートログ記録
```

### 3.4 発注処理フロー

```
1. 発注点チェック（在庫合計 ≤ reorder_point）
2. 混合グループチェック（グループ内全商品の状況確認）
3. order_pending へ INSERT
4. 保留期間経過または手動送信で order_history へ移動
5. 仕入先メールアドレスへ発注メール送信
6. order_history.mail_sent = 1 に更新
7. 在庫引当（_deduct_stock）
8. order_receipts で受入追跡
```

### 3.6 バックアップ設計

#### メインバックアップ（Excel）

対象テーブル:
- 商品マスタ、在庫ロット、廃棄在庫
- 発注履歴、入庫記録
- CSV設定、メール宛先、ユーザー

処理方式: Excelシート別出力（ヘッダのみスタイル適用、データは無スタイル）

#### 売上履歴バックアップ（CSV）

- 処理方式: バックグラウンドスレッドで生成 → ポーリングで進捗表示 → 完了後ダウンロード
- 出力列: JAN, 商品名, 数量, 売上日, 取込ファイル, チェーンCD, 得意先名, 店舗CD, 店舗名, ROW_HASH, 作成日時
- 復元時: `ROW_HASH` をキーにON CONFLICT DO NOTHING で重複スキップ

|--------------|---------|------|

**start レスポンス例:**
```json

  "job_id": "a1b2c3d4e5f6",
  "total": 125000
}
```

**progress レスポンス例:**
```json
{
  "total": 125000,
  "done": 62000,
  "finished": false,
  "error": null
}
```

エンドポイント: `GET /csv/progress/<job_id>/stream`

イベント形式:
```json
// 開始
{"phase": "start", "file": "sales_20260405.csv", "total": 5000}

// 進捗
{"phase": "progress", "current": 2500, "total": 5000, "ok": 2490, "skip": 10, "err": 0}

// ファイル完了
{"phase": "done", "file": "sales_20260405.csv", "ok": 4990, "skip": 8, "err": 2, "detail": "..."}

{"phase": "finished", "results": [{"name": "...", "status": "ok", "detail": "..."}]}
```

## 7. パフォーマンス設計

| 施策 | 内容 |
|------|------|
| 予測キャッシュ | `forecast_cache` テーブルに計算結果を24時間保存 |
