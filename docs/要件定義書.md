# 在庫管理システム 要件定義書

**バージョン**: 1.0
**作成日**: 2026-04-05
**対象システム**: 在庫管理・自動発注システム（inventory_system）

---

## 1. システム概要

### 1.1 目的

問屋・卸売業者向けに、AI需要予測を活用した在庫管理・自動発注を実現する。CSV取込による売上データ連携、気象データを活用した季節需要補正、52週MDプランによる中長期計画管理を一元化する。

### 1.2 適用範囲

- 商品マスタ管理
- 在庫管理（ロット管理含む）
- 発注管理（自動発注・手動発注）
- CSV取込による売上データ連携
- AI需要予測・ABC分析
- 52週MD（マーチャンダイジング）計画
- 気象データ連携・気温感応度分析
- レポート・アラート
- バックアップ・復元
- ユーザー管理・権限管理

### 1.3 利用者

| 役割 | 説明 |
|------|------|
| 管理者（admin） | 全機能へのアクセス・ユーザー管理 |
| 一般ユーザー（user） | 権限付与されたページのみアクセス |

## 3. 非機能要件

### 3.1 性能要件

| 要件 | 基準値 |
|------|--------|
| 通常画面のレスポンス | 3秒以内 |
| CSV取込（1万行） | 60秒以内 |
| 需要予測計算 | バックグラウンド実行（最大10分）・キャッシュ24時間 |
| 売上履歴CSVエクスポート | ストリーミング（進捗表示） |
| DB接続プーリング | 最小2接続・最大10接続 |

### 3.2 信頼性要件

- スケジューラによる自動処理は実行ログを保存し、重複実行を防止する
- CSV取込は行ハッシュにより重複インポートを防止する
- 発注処理は発注履歴に記録し、二重発注を防ぐ

### 3.3 セキュリティ要件

| 要件 | 内容 |
|------|------|
| 認証 | セッションベース認証（Flask-WTF） |
| パスワード | PBKDF2-SHA256 + ランダムソルトによるハッシュ |
| CSRF対策 | 全POSTフォームにCSRFトークン |
| 権限制御 | ページ単位の13権限 + ロール（admin/user） |
| 初回ログイン | パスワード強制変更 |
| ブルートフォース | Flask-Limiterによるレート制限 |

### 3.4 運用要件

- Windows Server / Windows 11 上で動作
- PostgreSQLデータベースを使用
- Waitressによる本番WSGIサーバー動作
- .envファイルによる環境設定管理
- 自動スケジューラによる無人運転対応

## 4. システム制約

- Python 3.x + Flask フレームワーク

## 5. 用語定義

| 用語 | 説明 |
|------|------|
| JAN | 日本の標準商品バーコード（13桁または8桁） |
| 発注点（Reorder Point） | 在庫数がこの値を下回ったら発注を起こす閾値 |
| 安全在庫 | 需要変動・リードタイム変動に備える余裕在庫 |
| ABC分析 | 売上金額の累積比率によるA/B/Cランク分類 |
| ホルト・ウィンタース | トレンド・季節性を考慮した指数平滑予測法 |
| IQR法 | 四分位範囲を使った外れ値除去手法 |
| MAPE | 平均絶対パーセント誤差（予測精度指標） |
| 52週MD | 1年52週の週次販売計画（マーチャンダイジング） |
| 混合グループ | 複数商品をまとめて1ロット単位で発注する設定 |
| 行ハッシュ | CSV取込の重複排除に使うレコードの一意ハッシュ値 |
